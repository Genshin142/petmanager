# StatsModule 仪表盘重构实施计划 (2026-05-06)

## 目标
将现有的基础报表模块升级为具有专业视觉、多维交互能力的宠物店经营仪表盘。

## 核心变更点

### 1. UI 架构重构
- **主布局**：采用顶部 KPI 卡片 + 中间多维切换区 (QStackedWidget) + 底部明细表的垂直结构。
- **视觉风格**：对齐 SaaS 现代化风格，使用 12px 圆角、微阴影、品牌蓝 (#3b82f6) 强调色。
- **详情下钻**：实现从右侧滑出的 `DetailDrawer`，支持点击明细行后查看具体订单详情。
- **分类导航**：通过 Tab 按钮无缝切换财务、服务、库存、会员四个核心看板。

### 2. 数据层集成
- **财务**：汇总商品与服务流水，计算净利润。
- **服务**：实时计算寄养入住率与洗护单量。
- **库存**：从 `ProductDataManager` 拉取滞销与告急 SKU。
- **会员**：分析拉新趋势与储值沉淀。

### 3. 性能与安全
- **跨表查询**：底层使用异步数据库读取，避免界面卡顿。
- **内存管理**：使用 `std::shared_ptr` 管理临时报表对象。

## 路线图

- [ ] **Phase 1**: 更新头文件，定义多维结构与 UI 组件。
- [ ] **Phase 2**: 实现 `setupUI()`，完成 Dashboard 基础布局。
- [ ] **Phase 3**: 数据引擎联调，对接各业务 DataManager。
- [ ] **Phase 4**: QSS 注入与视觉微调。
