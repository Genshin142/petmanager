# PetManager 操作日志模块 (Operation Log) 设计文档

## 1. 目标与范围 (Purpose and Scope)
为 PetManager 提供一个专业级的日志审计与对账界面，替代原有的弹窗提示。主要针对财务核对和异常操作追溯，确保所有关键业务（如订单状态更改、库存调整）都具备极高的数据透明度和可追责性。

## 2. 界面设计与布局 (UI/UX Design)
采用 **“Master-Detail（分栏视图）”** 架构，与 `SystemSettingsDialog` 保持高度的一致性，遵循 `Data-Dense Dashboard` 设计规范：
- **整体配色**：延续系统主色调（Primary #1E40AF, Secondary #3B82F6），保持严谨的商务质感。
- **左侧列表区 (Master)**：
  - **顶部**：核心过滤条件，如“日期范围”、“操作员工”和“模块/分类”。
  - **中部**：紧凑型列表视图，仅展示日志摘要（如时间、操作人、核心动作）。
  - **底部**：标准分页控件（上一页、页码、下一页），提供明确的浏览边界感。
- **右侧详情区 (Detail)**：
  - 展示被选中日志的完整上下文。
  - 核心包含 **Diff 差异对比表**。

## 3. 数据流与差异对比 (Data Flow & Diff Rendering)
- **数据加载机制**：采用数据库级分页。通过向 MySQL 发送带有 `LIMIT` 和 `OFFSET` 的查询请求，极大程度降低全量数据带来的内存开销和网络延迟。
- **底层数据结构**：日志详情字段（`operationLog`）采用结构化 JSON 格式存储。
  - 示例：`{"field": "订单状态", "old": "待接单", "new": "已完成"}`
- **视觉渲染方案**：
  - 客户端读取到 JSON 后，通过 `QJsonDocument` 进行解析。
  - 生成类似代码比对的视觉效果：**旧值 (Old Value)** 采用浅红色背景配合删除线；**新值 (New Value)** 采用浅绿色背景高亮，直观呈现数据变更走向。

## 4. 异常处理与兼容性 (Error Handling)
- **JSON 解析失败或兼容旧数据**：若日志详情内容为空、非标准 JSON 或旧版纯文本日志，右侧详情区将优雅降级为纯文本/富文本块展示，不会导致程序崩溃。
- **空状态占位 (Empty State)**：当没有选中任何左侧日志，或当前筛选条件下无数据时，右侧提供专业的水印背景或“暂无数据/请选择日志”提示。

## 5. 验证与测试 (Testing Strategy)
- 测试分页临界状态（如总页数为 0、1 或正好是分页单位倍数时，下一页/上一页按钮的可用状态）。
- 构造各种复杂的 JSON Diff 数据，验证 UI 层面的红绿背景和对齐效果是否错乱。
- 模拟数据库返回残缺数据，测试向后兼容的降级渲染是否正常。
