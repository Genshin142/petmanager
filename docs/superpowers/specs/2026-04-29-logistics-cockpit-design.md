# 规格说明书：动态双栏指挥舱 (Dynamic Dual-Lane Cockpit)

本规格说明书定义了 PetManager 车辆调度中心的重构方案。旨在解决多任务场景下的视觉混乱，提供跨日期视野，并引入专业级详情抽屉。

## 1. 核心视觉与交互设计

### 1.1 顶部：无限日期滚动条 (Infinite Date Scroller)
*   **交互逻辑**：支持左右滑动/点击切换。
*   **视觉锚定**：
    *   **选中态**：当前选中日期背景为蓝色圆角胶囊。
    *   **压力感应**：日期数字下方带有微型圆点。颜色代表当日任务密度（浅灰->浅蓝->橙色）。
    *   **回位功能**：左侧提供一个“今”图标按钮，点击瞬间回到今日视图。

### 1.2 中部：$N + 1$ 双栏看板 (Dual-Lane Kanban)
*   **布局**：左栏显示所选日期 $D$ 的任务，右栏显示 $D+1$ 的任务。
*   **排序**：卡片按 `appointmentTime` 从早到晚排序。
*   **卡片设计**：
    *   **彩色标识条**：左侧 4px 边框色（橙色：接宠；蓝色：送宠）。
    *   **选中态**：点击后边框加粗至 8px，卡片背景增加浅蓝色渐变底晕。
    *   **流水号**：右上角显示任务 ID（如 `#T1008`）。

### 1.3 右侧：详情抽屉 (Side Drawer)
*   **动画**：从界面右侧平滑滑出，宽度固定。
*   **内容密度**：
    *   **页眉**：圆形头像 + 昵称 + 业务标签。
    *   **勋章**：头像下方展示“疫苗状态”、“会员等级”徽标。
    *   **区块**：宠物档案、主客关系（含拨号按钮）、备注信息。
*   **操作流**：底部提供醒目的“确认完成”及“修改/取消”按钮。

---

## 2. 技术路线 (Qt/C++)

### 2.1 模块重构
*   **布局管理**：使用 `QHBoxLayout` 作为主容器。左侧为 `QScrollArea`（装载双栏看板），右侧为 `DetailPanel`（抽屉）。
*   **动画实现**：使用 `QPropertyAnimation` 对 `DetailPanel` 的 `minimumWidth` 或 `maximumWidth` 进行插值。

### 2.2 数据联动
*   **Manager 层**：`LogisticsManager` 需提供按日期过滤任务的接口。
*   **信号槽**：`DateScroller` 发出 `dateChanged(QDate)` 信号 -> 看板刷新 -> 清空详情抽屉。

---

## 3. 验收标准
1.  **看板清晰度**：能够一眼区分今日与明日的任务，无视觉重叠。
2.  **导航流畅度**：点击日期滚动条，下方看板能无感切换。
3.  **抽屉稳定性**：打开抽屉时，左侧看板布局不应发生剧烈晃动或重排。
4.  **业务闭环**：在抽屉中点击“完成”后，卡片状态同步更新，并记录宠物活动日志。
