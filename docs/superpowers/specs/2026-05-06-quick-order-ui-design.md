# 快速开单 (Quick Order) 界面设计规格书

## 1. 概述
本文档旨在定义“快速开单”界面的功能结构与视觉表现，以恢复并优化丢失的结算功能。界面采用高效的单页 POS 模式，旨在减少操作路径，提升到店客人的结算速度。

## 2. 界面结构 (三段式 + 侧边栏)

### 2.1 顶部：分类筛选区 (Category Bar)
- **功能**：横向排布的按钮组，用于切换商品/服务分类。
- **分类示例**：洗护、美容、寄养、零售、其他。
- **交互**：点击分类按钮，中间区域即时刷新对应的项目卡片。

### 2.2 中间：项目展示区 (Item Grid)
- **布局**：响应式网格布局。
- **卡片内容**：
    - 图标/图片 (Emoji 或图标占位)
    - 名称
    - 价格 (醒目蓝色显示)
- **交互**：点击卡片，对应项目自动添加到下方的待选列表中。

### 2.3 底部：已选清单区 (Selection Table)
- **布局**：表格形式显示。
- **字段**：项目名称、单价、数量 (带 +/- 调节)、小计、删除按钮。
- **功能**：支持实时修改数量和移除错误添加的项目。

### 2.4 右侧：结算侧边栏 (Checkout Sidebar)
- **会员管理**：
    - 快速搜索框：支持通过姓名/手机号关联会员。
    - 关联后显示：会员头像、姓名、当前余额/等级。
- **支付方式**：
    - 网格化按钮组：微信支付、支付宝、现金、会员余额。
- **统计与操作**：
    - **合计金额**：特大字号显示总额。
    - **按钮组**：
        - `取消开单`：清空当前所有选择并关闭。
        - `结算下单`：生成正式订单并打印小票 (可选)。

## 3. 视觉规范
- **主色调**：`#409eff` (品牌蓝)
- **背景色**：`#f8fafc` (浅灰背景，白色卡片)
- **圆角**：统一 `12px`
- **字体**：微软雅黑/系统默认 Sans-serif

## 4. 数据交互逻辑
1.  **加载**：从 `ProductDataManager` 和 `ServiceDataManager` 获取所有可用项目。
2.  **选购**：维护一个临时的 `QList<CartItem>` 结构。
3.  **结算**：调用 `PetDataManager::createOrder()` 方法将购物车数据持久化到数据库。
