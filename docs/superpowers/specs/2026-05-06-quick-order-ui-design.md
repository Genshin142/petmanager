# 快速开单 (Quick Order) 界面设计规格书 (v2)

## 1. 概述
本文档旨在定义“快速开单”界面的功能结构与视觉表现。根据最新反馈，界面应保持**高度紧凑**，移除冗余的支付环节，并增强会员与宠物的关联逻辑。

## 2. 界面结构 (三段式 + 侧边栏)

### 2.1 顶部：分类筛选区 (Category Bar)
- **设计建议**：按钮尺寸调小，间距紧凑。
- **功能**：横向排布，用于切换商品/服务分类。

### 2.2 中间：项目展示区 (Item Grid)
- **设计建议**：卡片尺寸缩小（例如 120px 宽），图标与文字更紧凑。
- **交互**：点击卡片即时入库。

### 2.3 底部：已选清单区 (Selection Table)
- **设计建议**：缩减行高，作为次要展示区。

### 2.4 右侧：侧边栏 (Compact Sidebar)
- **宽度**：缩减至约 320px。
- **客户信息区**：
    - **双下拉框布局**：[会员选择] [宠物选择] (仅在有服务项时显示/启用)。
    - **宠物联动**：选择会员后，宠物下拉框自动加载该会员名下的所有宠物。
- **支付方式**：**移除** (此界面仅负责开单生成订单，不处理实时支付)。
- **操作按钮**：
    - `取消` (Outline) / `下单` (Solid Primary)。

## 3. 视觉规范
- **风格**：极简、紧凑、高信息密度。
- **圆角**：统一 `8px` (调小)。
- **字体**：13px/14px 为主。

## 4. 业务逻辑
1.  **开单流程**：选择商品 -> (可选) 关联会员/宠物 -> 点击“下单”。
2.  **主界面联动**：点击“下单”后，`CheckoutModule` 的订单列表必须立即刷新，新增一行订单，其状态标记为 **“待结算”**。
3.  **服务项限制**：若订单包含“洗护/美容”等服务，必须强制选择宠物。
