# CheckoutModule 订单中心优化设计方案

## 1. 背景与目标
目前 PetManager 的订单中心在数据统计和日期筛选上存在易用性痛点：
- 数据统计卡片随筛选变动，无法直观对比今日实时数据。
- 日期筛选仅支持手动选择，查询特定周期（如“本月”）操作繁琐。

本设计方案旨在通过“锁定核心指标”和“快捷日期预设”提升店员对账效率和店主经营管理体验。

## 2. 界面布局方案 (方案 A-1)
采用侧边集成式布局，将快捷操作与现有筛选栏融合。

### 2.1 统计仪表盘 (Cockpit Stats)
重构顶部统计卡片，采用三卡片设计：
- **卡片 1：今日实时营收**
    - **逻辑**：固定显示 `QDate::currentDate()` 的已支付订单总额。
    - **UI**：数字颜色 `#3b82f6` (蓝色)，增加一个绿色呼吸灯效果的小圆点，标注“实时”。
- **卡片 2：所选时段营收**
    - **逻辑**：随下方日期筛选器动态更新统计值。
    - **UI**：数字颜色 `#1e293b` (深灰)，作为查询结果的核心指标。
- **卡片 3：当前待处理订单**
    - **逻辑**：显示当前所有“待结算”状态的订单数量。
    - **UI (主)**：数量 > 0 时显示橙色并带有微弱闪烁效果；数量 = 0 时显示绿色勾选图标。
    - **UI (次)**：底部灰色小字显示“平均客单价: ¥XXX.XX”，通过细微水平线隔离。

### 2.2 日期快捷筛选器 (Date Shortcut)
在 `时段:` 标签左侧增加 5 个扁平化胶囊按钮：
- **预设项**：`[ 今天 ]` `[ 昨天 ]` `[ 本周 ]` `[ 本月 ]` `[ 上月 ]`
- **交互逻辑**：
    1. 点击按钮后，自动计算起始日期并填入 `m_startDateEdit` 和 `m_endDateEdit`。
    2. 自动触发 `onFilter()` 刷新列表。
    3. 按钮具有“激活”状态样式（背景色改为浅蓝，边框变深）。

## 3. 技术实现要点
### 3.1 统计逻辑
- 修改 `updateStats()` 方法，内部调用两次 `PetDataManager::getOrderStats`：
    - 一次传入当前日期用于“今日实时营收”。
    - 一次传入 `m_startDateEdit/m_endDateEdit` 用于“所选时段营收”。

### 3.2 日期计算工具类
实现一个辅助函数 `calculateRange(ShortcutType type)`：
- `今天`：Start=End=Today.
- `昨天`：Start=End=Today-1.
- `本周`：Start=Monday of this week, End=Today.
- `本月`：Start=1st of this month, End=Last day of this month.
- `上月`：Start=1st of last month, End=Last day of last month.

## 4. 成功标准
- 店员能在 1 秒内切换到“本月”查看业绩。
- 筛选任何历史数据时，今日营收依然清晰可见，不影响收银对账。
- “待处理订单”能有效警示未结账业务。
