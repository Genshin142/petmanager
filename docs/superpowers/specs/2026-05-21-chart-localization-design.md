# 2026-05-21 数据报表模块扇形图汉化设计方案

## 1. 目标与背景
数据报表模块中的“营收项目构成”和“支付渠道分布”两个扇形图（饼图）中的分类标签当前显示为英文（例如："Appointment"、"Product"），为了提升用户体验，需要将图表中的标签全部替换为规范的中文名称。

## 2. 设计方案
选择**方案一：客户端动态翻译映射**。
在客户端图表渲染时，对英文系统键值（键名来自于数据库及网络包中的 raw 字段）进行拦截，动态翻译为对应的中文标签，不改动数据库中的原始键名，确保对系统其他底层业务（如订单管理、统计流转等逻辑）零侵入。

### 2.1 汉化对照表

| 原始英文键 (数据库/网络层) | 翻译后中文标签 | 说明 |
| :--- | :--- | :--- |
| `Product` | `商品零售` | 商品订单来源 |
| `Appointment` | `服务预约` | 服务预约来源 |
| `Service` | `服务项目` | 增值服务来源 |
| `Foster` / `Boarding` | `宠物寄养` | 寄养业务来源 |
| `Logistics` | `宠物接送` | 派单接送来源 |
| `MemberCard` / `Balance` | `会员卡余额` | 支付渠道 |
| `Alipay` | `支付宝` | 支付渠道 |
| `Wechat` / `WeChat` | `微信支付` | 支付渠道 |
| `Cash` | `现金` | 支付渠道 |
| `Card` | `银行卡` | 支付渠道 |

### 2.2 具体修改点
- **修改文件**：`modules/statsmodule.cpp`
- **具体函数**：`StatsModule::onDashboardStatsReceived` 内部的本地 lambda 渲染函数 `updatePie`。
- **修改逻辑**：
  在循环提取 `obj["name"].toString()` 后，进行英文到中文的条件翻译转换，然后再将转换后的中文名称作为 `series->append(name, val)` 的参数传入。

## 3. 验证计划
1. **编译运行项目**，进入数据报表模块。
2. **观察图表**，确认“营收项目构成”和“支付渠道分布”扇形图中的分类指标完全显示为对应的中文名称，如“服务预约”、“商品零售”、“微信支付”等。
3. **核对比例与数值**，确保数值和百分比未受干扰且计算正确。
